AI that runs in your browser

SimpleUtils AI tools don't send your images or data to any AI API. The models download and run directly inside your browser.

How AI tools usually work

Most AI tools are a frontend that sends your data to a backend API — OpenAI, Replicate, or a private server. Your image or text is uploaded, processed in the cloud, and the result is returned. The AI never runs on your machine.

How SimpleUtils AI tools work

SimpleUtils uses WebAssembly and the browser's built-in compute capabilities to run real AI models directly on your CPU (and GPU where available). The model files download once from a public CDN and are cached in your browser — after that, they run offline with no internet required.

Your image or data is processed entirely in JavaScript/WASM inside the browser tab. Nothing is sent to a server.

Which tools use AI?

🎭
Background Remover
Uses @imgly/background-removal — a WASM-powered segmentation model that runs in your browser. Model size: ~40MB, cached after first use.
🎯
Subject Keeper
Uses CLIPSeg via @xenova/transformers — a text-prompted image segmentation model from Hugging Face. Model size: ~300MB, cached after first use. Runs on your CPU via WebAssembly.

Why does it take a minute the first time?

The model files (neural network weights) need to download the first time you use a tool. Once downloaded, your browser caches them — subsequent uses are instant and work even without an internet connection.

Typical model sizes:
Background Remover~40 MB
Subject Keeper (CLIPSeg)~300 MB

What technology makes this possible?

  • WebAssembly (WASM) — lets compiled code (like neural network inference engines) run at near-native speed in the browser
  • Transformers.js — Hugging Face's JavaScript port of their popular ML library, enabling hundreds of models to run client-side
  • ONNX Runtime Web — a cross-platform ML runtime that works in the browser
  • WebGPU / WebGL — where your browser supports it, GPU acceleration is used automatically
